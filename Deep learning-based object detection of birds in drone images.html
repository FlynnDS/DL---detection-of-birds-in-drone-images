Deep Learning 2025 Group Assignment

Deep learning-based object detection of birds in drone images

Scarecrow is a collaborative project between Goossens Malderen (a farm specializing in white asparagus), Avular (drone technology), Qualified, and Datacation (data science). The project is supported by Interreg, a European program that funds regional cooperation.

 

Challenge

Birds can cause damage to crops such as peas, beans, grapes and young vegetable plants. In practice, farmers often use bird cannons, scarecrows or hunting to scare birds away. While these methods can help, they also have limitations. Birds may become used to them, making them less effective over time, and some methods (like cannons) create noise disturbance for nearby residents.

The Scarecrow project aims to create a smarter, more sustainable solution by combining drone technology with motion detection and automated deterrent systems. The projects has two goals: 1) developing a model that uses computer vision to automatically detect birds in a field, and 2) creating a system to control drone flight paths to avoid birds getting used to the behavior.

Objective

This group assignment focuses on the first step: detecting birds in images captured by drones. Your objective is to develop an object detection model capable of accurately identifying and localizing birds within these images. You are expected to complete the entire pipeline, from loading and preprocessing the dataset, selecting and training a model, to evaluating its performance and conducting error analysis.

Your code should be reproducible and work on the computer of the teacher and data consultant using your trained network weights.

   

Important requirements

In a real-world setting like this, it is important to build a solution that is practical and easy to deploy rather than to aim for the most advanced or complicated model. While a highly complex model may achieve better results on paper, it could also be too slow, resource-intensive or difficult to integrate into an operational system. As the system is ultimately meant to detect and scare away birds in real-time, an important requirement of your model is to be fast and efficient, and not just accurate. You will also be working with a relatively small dataset, so you are encouraged to explore techniques data augmentation or supplement your training data with relevant open-source datasets. Given that birds often appear small in drone images, your model should be capable of detecting small objects. Furthermore, keep in mind that detecting sitting birds is more important than detecting those in flight, as they are the ones causing damage to the crops. When evaluating your model, consider the trade-off between precision and recall: would it be better to detect every bird, even if some predictions are wrong, or to only detect birds when the model is almost certain? Think about what would be most useful in practice.

Of course, you don’t need to tackle every challenge perfectly. However, your approach should show that you have thought about the requirements in the choices and design of the solution.

 

Data

The provided dataset contains 326 images captured by drones. The dataset is already split into a training set (263 images), validation set (32 images) and an independent test set (31 images). All images have their annotations in the corresponding .json file. The images are annotated using a single label ‘bird’. No distinction is made between bird species (e.g., crows, pigeons, etc.). An example notebook to visualize these annotations on images is included (visualizing_labels.ipynb).

Grading

The final grade is based on the following components:

  • Code (40%) – 4 points max
  • Report (30%) – 3 points max
  • Presentation (30%) – 3 points max

Code (40%) - 4 points max:

  1. Data loading and preprocessing
  • Correct implementation of data loaders for images and annotations for your specific object detection model
  • Use of data augmentation techniques
  • Appropriate shuffling  and batching of data
  • Conduct an online search for relevant open-source datasets, and if you can find them, use them in your application as additional training data (to improve generalization)

  1. Model approach and methodology
  • Selection of a suitable object detection model for small objects. Some of the model that are currently available in open-source: YOLO, Faster R-CNN, RetinaNet, CenterNet, RF-DETR.
  • Application of regularization techniques (e.g., dropout, batch normalization)
  • Hyperparameter tuning (learning rate, schedulers)
  • Think of better loss functions for small object detection and implement them if possible (e.g. focal loss)
  • Appropriate choice of confidence threshold, and Non-Maximum suppression.
  • Appropriate use of evaluation metrics that are easy to interpret for the teacher and data consultants (e.g., precision/recall, F1-score, AP, mAP)

  1. Code quality and reproducibility
  • Ensure reproducibility (e.g., set random seeds)
  • Clean, well-structured and properly commented code
  • Write a README file for running your code in inference mode to produce the targets. This will help the teacher and the data consultatns to reproduce your results

 

Report (30%) – 3 points max:

In the written report (maximum 5 pages), include the following:

  • A clear summary of the approach
  • Justification of decisions in relation to the specific use case, such as:
    • Choice of model; applicable for detection of small objects, and being fast for drone deployment
    • If applicable, the use data augmentation techniques
    • The choice of evaluation metrics and thresholds for confidence and non-maximum suppression. Do they lead to a high precision or high recall, and what’s better for this use case?
  • Reflection on the results:
    • Any trade-offs made or difficulties encountered
    • How the model performance relates to the real-world application and deployment on drone (analysis speed, memory footprint of model)
    • Measure and report performance (e.g., frames per second)
    • Suggestions for future improvements (you can use references to scientific papers or Github repositories)
  • Please be concise in your reporting (max. 5 pages): exceeding 5 pages will result in a grade deduction, meaning that you will not receive the full 3 points for section 'report'.

Presentation (30%) – 3 points max:

The student presentation should be 5 minutes followed by 5 minutes Q&A with the teacher and data consultants of Datacation. Please don’t introduce the subject in your presentation to keep it compact. Ideally, your presentation should:

  • Clearly communicate chosen approach to the problem
  • Highlight the design choices and innovations
  • Summarize results and justify their relevance
  • Show clear understanding of the task, taking into account the hardware constraints (fast deployment, small memory footprint model).
  • Make the connection to lessons learned in the course

 

Submit these files

  1. Python code (.py or .ipynb)

Computing hardware and tips for training

This assignment involves training images with an object detection model, and this requires appropriate computing hardware, preferably a CUDA-supported GPU. Select your group members such that at least one of you has a computer with CUDA-supported GPU. In the situation that you or your group members have trouble training, please use alternative cloud computing options, like Google Colab or Amazon AWS.

Some Colab examples  and videos tutorials can be found here:

  • YOLOv11 in Google Colab
  • Various YOLO models in Google Colab
  • Detectron2 in Google Colab

In the worse-case scenario that you cannot properly train the object detection model, please contact the teacher (p.m.blok@tilburguniversity.edu) and he’ll provide you an alternative assignment for time-series analysis that requires less computing resources to train. Contact him as quickly as possible!
